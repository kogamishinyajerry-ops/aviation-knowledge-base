### 4.3 Provenance / Confidence / Source contracts

Three structured objects mandatory on every record:

- `provenance` — `method` enum (human / ai_extracted / hybrid_reviewed), `actor`, `actor_role`, `created_at`, optional `reviewer`, `reviewed_at`, `tool` (e.g. `claude-opus-4-7`)
- `confidence` — `score` (0.0–1.0), `rationale` (≥1 sentence; enum from ADR-005), optional `calibration_method`
- `source` — `document_id` (URI to Document entity), `locator` (page/section/paragraph), `retrieval` (url, retrieved_at, content_hash), `effective_date`

**H-Darrieus lock (PROV-04, locked in ADR-005):** A record with `provenance.method=ai_extracted` AND `confidence.score>0.85` AND no `reviewer` is REJECTED at validation time. `instances/_pending/` is the AI-extraction quarantine; canonical promotion requires `provenance.method=hybrid_reviewed` + populated reviewer + reviewed_at.

### 4.4 Schema versioning

`ontology/VERSION` = semver; per-record `schema_version`; `migrations/` directory pattern; `ontology/CHANGELOG.md` records every change with rationale.

## 5. Validators

`python scripts/validate.py` runs in single pass:

1. **schema validation** (`scripts/validators/schema.py`) — JSON Schema 2020-12 against entity + relation schemas
2. **id format check** (`scripts/validators/ids.py`) — URI scheme `aviationkb://<type>/<slug>@<version>`
3. **relation referent existence** (`scripts/validators/relations.py`) — subject + object entities resolve
4. **provenance gate** (`scripts/validators/provenance.py`) — H-Darrieus REJECT, _pending promotion gate, schema_version
5. **link check** (`scripts/validators/links.py`) — broken-ref + supersession integrity (no cycles, no breaks)

Test fixtures: `tests/fixtures/{valid,invalid}/` covering every entity type, relation type, _pending promotion path, supersession chain, ai_extracted-without-reviewer rejection.

CI: GitHub Actions runs `validate.py` + `pytest` on every push/PR; blocks merge on failure.
